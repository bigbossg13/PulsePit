## Overview

The command-based framework separates robot code into two core abstractions: **Subsystems** and **Commands**.

## Subsystems

A `Subsystem` represents a physical mechanism on the robot — a drivetrain, arm, or shooter. Each subsystem owns its hardware and exposes methods that commands call.

```java
public class Arm extends SubsystemBase {
  private final CANSparkMax m_motor = new CANSparkMax(1, MotorType.kBrushless);

  public void setSpeed(double speed) {
    m_motor.set(speed);
  }

  @Override
  public void periodic() {
    // Called every loop; update SmartDashboard here
  }
}
```

## Commands

A `Command` describes what a subsystem should do and when it is finished.

```java
public class RaiseArm extends CommandBase {
  private final Arm m_arm;

  public RaiseArm(Arm arm) {
    m_arm = arm;
    addRequirements(arm);
  }

  @Override
  public void execute() { m_arm.setSpeed(0.5); }

  @Override
  public boolean isFinished() { return m_arm.atSetpoint(); }

  @Override
  public void end(boolean interrupted) { m_arm.setSpeed(0); }
}
```

## The Scheduler

`CommandScheduler.getInstance().run()` is called every loop (≈20 ms). It:

1. Polls triggers and buttons
2. Runs each scheduled command's `execute()` method
3. Checks `isFinished()` and ends commands that return `true`

## Button Bindings

Bind commands to controller buttons in `RobotContainer`:

```java
new JoystickButton(m_controller, Button.kA.value)
  .whileTrue(new RaiseArm(m_arm));
```
